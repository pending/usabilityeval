Error Prevention

Even better than good error messages is a careful design which prevents a problem from occurring in the first place. Either eliminate error-prone conditions or check for them and present users with a confirmation option before they commit to the action.

The website incorporates elements of intuitive user behaviors to dodge and avoid potential errors common to users, web programmers, and news sources alike. The system includes within its domain additional addresses that could otherwise potentially lead users to an error due to obsolete locations or web incompatibility. Common repositories that might suggest a certain location in the system (such as the first key example), but in fact are actually inactive are kept as live addresses and serve as an efficient method of redirection of user traffic. The second key example demonstrates a great case of error prevention by the system in the manner it was able to address human error and account for typos by incorporating increased intuition in address localities. The system adapts from the misspelled link an intuitive suggestion for a possible destination of the user -- a “/hi” extension resembles the shortcut or typo to the “/history” address -- and avoids potential abruptions to an otherwise pleasant experience. The system provides the website through a thoughtful presentation considering the agile multi-platform compatibility allowing even mobile users ease of access to popular pages. On the other hand, the search bar does not prevent blank queries, and simply shows an empty page that does not notify the user what is wrong. Some form of prevention (such as a pop-up saying no empty searches) would help this problem.
